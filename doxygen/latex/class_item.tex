\doxysubsection{Detailed Description}
Class represents an item that can be worn by a character. 

Game Rules\+: Each item can have one or more enhancements associated with it, such as damage bonus, attack bonus, or increased attributes. Different types of items (\doxyref{Helmet}{p.}{class_helmet}, \doxyref{Armor}{p.}{class_armor}, \doxyref{Shield}{p.}{class_shield}, etc.) provide different sets of enhancements. \doxyref{Item}{p.}{class_item} containers (\doxyref{Backpack}{p.}{class_backpack}, \doxyref{Worn\+Items}{p.}{class_worn_items}, \doxyref{Treasure\+Chest}{p.}{class_treasure_chest}) allow players to manage and store their items.

Design\+: The design follows a hierarchical structure with base class \doxyref{Item}{p.}{class_item} representing all items in the game. Derived classes like \doxyref{Helmet}{p.}{class_helmet}, \doxyref{Armor}{p.}{class_armor}, \doxyref{Shield}{p.}{class_shield}, etc., represent specific types of items with specialized functionalities. \doxyref{Item\+Container}{p.}{class_item_container} is a base class representing containers for items, allowing addition, removal, and display of items. Derived classes like \doxyref{Backpack}{p.}{class_backpack}, \doxyref{Worn\+Items}{p.}{class_worn_items}, etc., specialize the container behavior. The code uses polymorphism to handle different item types uniformly through base class pointers. Utility function enhancement\+To\+String converts enhancement enum values to their string representations.

Libraries\+: $<$vector$>$ for storing the map grid due to its flexibility in handling dynamic 2D arrays. $<$string$>$ from the C++ Standard Library for managing and manipulating string data
